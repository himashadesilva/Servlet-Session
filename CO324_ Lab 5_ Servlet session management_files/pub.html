CO324 Lab 05: Session Management

21 Feb.

Outcomes

At the end of this lab you should be able to,

  • Recognize the need for HTTP sessions.
  • Use the Servlet session API to manage HTTP sessions.

References

  • http://www.docstore.mik.ua/orelly/java-ent/servlet/ch07_05.htm
  • http://www.javamex.com/tutorials/servlets/session_api.shtml
  • http://www.javatpoint.com/http-session-in-session-tracking

Exercises

These exercises are an extension of the address book you implemented in the previous lab.

  1. Combine the doGet handler for search and the doPost handler for adding new entries into a single servlet.
  2. Create a separate session when a user accesses the address book servlet the first time.
  3. Verify that each user gets their own address book using two private tabs in the browser. That is, entries added by one user should not be visible to another user.
